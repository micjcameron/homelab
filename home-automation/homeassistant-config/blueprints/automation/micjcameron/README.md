# My automation blueprints

Author your own reusable automation blueprints here. This folder **is** version
controlled and deploys to the Pi (unlike HA's built-in `homeassistant/` examples
and HACS `custom_components/`, which stay on the Pi).

## Layout

```
blueprints/
├── automation/
│   ├── homeassistant/      # HA built-in examples — gitignored, regenerated by HA
│   └── micjcameron/        # ← your automation blueprints (tracked + deployed)
└── script/
    └── micjcameron/        # ← your script blueprints (tracked + deployed)
```

## Using one

A blueprint's `path` is relative to `blueprints/automation/`. So a file
`micjcameron/motion_light.yaml` is referenced from
[../../../automations.yaml](../../../automations.yaml) as:

```yaml
- alias: Hallway motion light
  use_blueprint:
    path: micjcameron/motion_light.yaml
    input:
      motion_entity: binary_sensor.hallway_motion
      light_target: { entity_id: light.hallway }
```

## Workflow

```bash
# 1. Author/edit a blueprint here on the Mac
# 2. Commit & push
git add . && git commit -m "add motion_light blueprint" && git push
# 3. Deploy (copies to the Pi, validates, restarts HA)
./home-automation/scripts/ha-deploy.sh
```

The deploy restart picks up new/changed blueprints. (Editing only an automation
that *uses* a blueprint reloads without a full restart, but a blueprint
definition change needs the restart that `ha-deploy.sh` already does.)
